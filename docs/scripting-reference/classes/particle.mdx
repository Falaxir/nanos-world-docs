# ‚ú® Particle

Particle Entity.

import { Structs, BasicType, AuthorityType, AssetPath, Classes } from '@site/docs/components/_nanos.mdx';

![](/img/docs/particle.jpg)

:::info

üü¶üüß **Authority:** This class can be spawned on both [Client](./core-concepts/scripting/authority-concepts.mdx) and [Server](./core-concepts/scripting/authority-concepts.mdx) (if you spawn it on client, it won't be synchronized with other players).

‚è¨ **Inheritance:** This class inherits from <Classes.Actor />.

:::

:::tip

Both **Cascade** and **Niagara** Particle Systems are supported!

:::


## Usage


```lua
local my_particle = Particle(
    Vector(-200, 100, 100),
    Rotator(0, 0, 0),
    "nanos-world::P_Explosion",
    true, -- Auto Destroy?
    true -- Auto Activate?
)
```

## Constructor Parameters

| **Type** | **Name** | **Default** |
| :--- | :--- | :--- |
| <Structs.Vector /> | **`location`** | `Vector(0, 0, 0)` |
| <Structs.Rotator /> | **`rotation`** | `Rotator(0, 0, 0)` |
| <AssetPath.Particle /> | **`asset`** | ` ` |
| <BasicType.Boolean /> | **`auto_destroy`** | `true` |
| <BasicType.Boolean /> | **`auto_activate`** | `true` |

:::info Note

**`auto_destroy`** means the Entity will be immediately destroyed after spawned, losing references to the _Particle System_ spawned in-game. So if the _Particle System_ itself loops indefinitely, it will keep playing until the Player reconnects.

:::
